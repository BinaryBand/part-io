# part-io

**part-io** is an audio jingle-matching toolkit that locates and extracts recurring jingles/stingers from long episode recordings. It combines **spectral-feature matching** with a **human-in-the-loop workflow** for precision.

### Key Features
- **Cold-start discovery**: Interactively narrows down jingle regions via tile-based bisection.
- **Spectral matching**: Finds jingles across episodes using reference samples.
- **Interactive CLI**: A guided terminal workflow with **Typer** and **Rich** for seamless argument input.
- **Modular design**: Built with Typer for flexibility and architecture guardrails.

### Terminal User Interface (TUI) Stack
The CLI leverages the following TUI packages for an intuitive experience:

- **Typer**: The core CLI framework for command registration, argument parsing, and context handling.
- **Rich**: Provides styled prompts, progress bars, and interactive menus for a polished UI.
- **Questionary**: Arrow-key navigation for menus (falls back to Rich prompts in non-TTY environments).

### Terminal Interface Walkthrough
The CLI offers two modes:

1. **Interactive Picker**: Run `partio` to select commands from a numbered menu. The tool **automatically prompts for required arguments** (e.g., file paths, thresholds) with type-aware Rich UI.

2. **Direct Mode**: Invoke commands with flags (e.g., `partio audio bootstrap --source episode.mp3`). Missing arguments trigger **fallback prompts** for clarity.

#### Workflow Steps
1. **Bootstrap**: Extract a seed clip from an episode region.
2. **Search/Locate**: Find jingle matches across episodes.
3. **Review**: Extract clips for manual verification.

Ideal for audio researchers, podcasters, and developers who need **reliable jingle extraction** with minimal setup.
